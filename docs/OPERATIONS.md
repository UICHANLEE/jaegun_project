# 운영 체크리스트

## 행사 전날

- Vercel 배포 URL 접속 확인
- `UPSTASH_REDIS_REST_URL`, `UPSTASH_REDIS_REST_TOKEN` 설정 확인
- 관리자 페이지에서 `레크레이션 화면으로 되돌리기` 클릭 후 참가자 화면 초기 상태 확인
- 휴대폰 3대 이상으로 전환 테스트
- 힌트코드 12개 입력 테스트
- Wi-Fi 신호가 약한 공간에서 참가자 페이지 새로고침 테스트

## 행사 30분 전

- 관리자 기기 1대: `index.html`
- 참가자 안내 QR: `player.html`
- 모든 참가자 화면이 밝은 레크레이션 화면인지 확인
- 관리자 상태 표시가 `공유 서버`인지 확인
- `임시 서버` 또는 `이 브라우저 미리보기`로 표시되면 Redis 설정을 다시 확인

## 전환 순간

1. 진행 멘트로 레크레이션 분위기를 유지합니다.
2. 관리자 페이지에서 `크라임씬으로 전환`을 클릭합니다.
3. 1~2초 안에 참가자 화면이 어두운 크라임씬 UI로 바뀌는지 확인합니다.
4. 늦게 바뀌는 팀은 새로고침을 안내합니다.

## 장애 대응

| 증상 | 원인 후보 | 조치 |
| --- | --- | --- |
| 일부 휴대폰만 전환 안 됨 | 네트워크 지연, 절전 브라우저 | 새로고침 안내 |
| 모든 휴대폰 전환 안 됨 | API/Redis 설정 오류 | 관리자 상태 표시 확인, Vercel 환경 변수 확인 |
| 관리자에는 전환됐는데 참가자 변화 없음 | 캐시 또는 다른 배포 URL 사용 | QR URL과 관리자 URL이 같은 배포인지 확인 |
| 전환이 되돌아감 | 다른 관리자 기기가 되돌리기 클릭 | 관리자 기기 1대만 사용 |

## 권장 운영 규칙

- 관리자 페이지 URL은 진행자만 접근합니다.
- 참가자 QR에는 `player.html`만 넣습니다.
- 리허설에서는 5대 이상, 본행사 전에는 가능한 실제 25대에 가깝게 테스트합니다.
- 현장에서는 관리자 기기를 충전기에 연결하고 화면 꺼짐을 방지합니다.
